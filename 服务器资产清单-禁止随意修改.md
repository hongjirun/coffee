## 🚫 严禁操作清单

### 1. 禁止修改 MySQL root 密码
- 咖啡店直接用 root/123456 连接，改了会宕机

### 2. 禁止强杀 MySQL 进程
- 严禁 `pkill -f mysqld` 或 `kill -9 [mysql进程]`
- 会导致 InnoDB 数据文件损坏，数据全部丢失（血泪教训）
- 正确停止：`/etc/init.d/mysqld stop`

### 3. 禁止占用已有端口
- 3000/3001/3002/3003/3005 全部已占用

### 4. 禁止删除以下目录
- `/www/wwwroot/coffee-api/`
- `/www/wwwroot/admin.xianshihuodong.xyz/`
- `/www/wwwroot/what-to-eat-backend/`
- `/www/jintianiyoujuma/`
- `/www/amz/`
- `/opt/parking/`
- `/www/server/data/`（MySQL数据目录）

### 5. 禁止重装宝塔 MySQL
- 会清空所有数据库

### 6. 修改 Nginx 前必须备份
```bash
cp /www/server/panel/vhost/nginx/xxx.conf /www/server/panel/vhost/nginx/xxx.conf.bak
nginx -t && /etc/init.d/nginx reload
```

## 新项目部署规范

### 部署前检查
```bash
pm2 status                          # 确认现有进程
netstat -tlnp | grep node           # 确认端口占用
mysql -u root -p'123456' -e "SHOW DATABASES;"  # 确认数据库
```

### 备份现有数据库（每次部署前必做）
```bash
mysqldump -u root -p'123456' xianshihuodong > /tmp/coffee_$(date +%Y%m%d).sql
mysqldump -u root -p'123456' parking_db_v2 > /tmp/parking_$(date +%Y%m%d).sql
```

### 新项目数据库规范
```bash
# 创建独立用户，不用 root
mysql -u root -p'123456' -e "
  CREATE DATABASE 新项目_db CHARACTER SET utf8mb4;
  CREATE USER '新项目_user'@'localhost' IDENTIFIED BY '独立密码';
  GRANT ALL ON 新项目_db.* TO '新项目_user'@'localhost';
  FLUSH PRIVILEGES;
"
```
