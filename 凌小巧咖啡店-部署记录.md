## 🚫 禁止操作清单（新项目部署时严禁触碰）

### 1. 禁止修改 MySQL root 密码
- 当前 root 密码为 `123456`，咖啡店后端直接使用 root 连接
- **修改 root 密码会导致咖啡店 API 全部 500 错误**
- 新项目请创建独立数据库用户：
  ```bash
  mysql -u root -p'123456' -e "CREATE USER '新用户'@'localhost' IDENTIFIED BY '新密码'; GRANT ALL ON 新数据库.* TO '新用户'@'localhost';"
  ```

### 2. 禁止强杀 MySQL 进程
- 严禁使用 `pkill -f mysqld` 或 `kill -9` 杀 MySQL
- 上次强杀导致 InnoDB 数据文件损坏，数据全部丢失
- 正确停止方式：`/etc/init.d/mysqld stop`

### 3. 禁止占用 3003 端口
- 3003 端口已被咖啡店后端占用
- 新项目请使用 3005 及以后的端口（3004 被停车场占用）

### 4. 禁止删除或覆盖以下目录
- `/www/wwwroot/coffee-api/` — 咖啡店后端
- `/www/wwwroot/admin.xianshihuodong.xyz/` — 咖啡店管理后台
- `/www/server/data/` — MySQL 数据目录（所有项目共用）

### 5. 禁止重装或降级宝塔 MySQL
- 当前版本：MySQL 5.7（宝塔管理）
- 重装会清空所有数据库

### 6. 禁止随意 reload/restart Nginx 配置文件
- 修改 Nginx 前先备份：`cp 原文件 原文件.bak`
- reload 前检查语法：`nginx -t`

## 新项目部署规范

### 部署前必做
1. 运行 `pm2 status` 确认现有进程
2. 运行 `netstat -tlnp | grep 300` 确认端口未被占用
3. 备份数据库：
   ```bash
   mysqldump -u root -p'123456' xianshihuodong > /tmp/coffee_backup_$(date +%Y%m%d).sql
   ```

### 数据库规范
- 每个新项目创建独立数据库用户，**不使用 root**
- 示例：
  ```bash
  mysql -u root -p'123456' -e "
    CREATE DATABASE 新项目db CHARACTER SET utf8mb4;
    CREATE USER '新项目_user'@'localhost' IDENTIFIED BY '独立密码';
    GRANT ALL ON 新项目db.* TO '新项目_user'@'localhost';
    FLUSH PRIVILEGES;
  "
  ```

### 端口规范
- 新项目从 **3005** 开始往后递增使用
